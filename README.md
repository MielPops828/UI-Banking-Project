# Тест-кейсы для UIBankingProject

## Объект тестирования
https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager

## Технологии:
- Java 17;
- Selenium WebDriver;
- TestNG;
- Maven;
- Allure Reports.

## Тест-кейс №1. Создание клиента (Add Customer)
### Шаги:
1. Открыть браузер (Google Chrome);
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager;
3. Открыть вкладку "Add Customer";
4. Сгенерировать номер и записать его в поле Post Code;
5. Для поля First Name сгенерировать имя на основе Post Code;
6. Заполнить поле First Name сгенерированным значением;
7. Заполнить поле Last Name;
8. Нажать на кнопку "Add Customer".

### Ожидаемый результат:
- Появился алерт, подтверждающий успешное добавление клиента.

## Тест-кейс №2. Сортировка клиентов по имени (First Name)
### Шаги:
1. Открыть браузер (Google Chrome);
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager;
3. Открыть вкладку "Customers";
4. Получить список всех First Name из таблицы;
5. Нажать кнопку сортировки/заголовок колонки First Name;
6. Сравнить порядок, полученный на UI, с алфавитно отсортированным списком.

### Ожидаемый результат:
- Порядок имён в таблице совпал с алфавитно отсортированным списком.

## Тест-кейс №3. Удаление клиента по длине имени (ближайший к среднему)
### Шаги:
1. Открыть браузер (Google Chrome);
2. Перейти по ссылке https://www.globalsqa.com/angularJs-protractor/BankingProject/#/manager;
3. Открыть вкладку "Customers";
4. Получить список всех First Name из таблицы;
5. Вычислить длину каждого имени;
6. Вычислить среднее арифметическое длин имен;
7. Найти имя, длина которого ближе всего к среднему;
8. Удалить клиента по найденному имени по нажатию кнопки Delete.

### Ожидаемый результат:
- Клиент с именем, длина которого ближе к среднему, удалён из таблицы.
